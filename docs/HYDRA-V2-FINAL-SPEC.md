# HYDRA v2 최종 사양서

> 이 문서가 HYDRA v2의 유일한 진실(source of truth)입니다.
> 모든 코드는 이 문서에 맞춰야 합니다.
> 작성일: 2026-04-17

---

## 1. 프로젝트 목표

YouTube 영상에 자연스러운 댓글 마케팅을 수행하는 멀티머신 협업 패널.
타겟 키워드 관련 **모든 영상**에 댓글을 달아서 전국민이 인지하게 만든다.

### 핵심 원칙
- 모든 행동은 사람처럼. 고정값/패턴 절대 금지. 가우시안 랜덤.
- 질보다 양, 오래가는 것이 최우선.
- 화질 건드리지 않음. 시청 시간으로 데이터 관리.
- 배포 후 다시 수정하지 않게 완벽하게.

## 2. 아키텍처

```
클라우드 VPS:
├── FastAPI 서버 (API + WebSocket)
├── PostgreSQL
├── React 대시보드 (Nginx 서빙)
└── Docker Compose로 배포

Worker PC (N대, Windows/Mac):
├── Python Worker 앱
├── AdsPower (동일 계정 로그인)
├── Playwright (CDP 연결)
├── 휴대폰 USB 테더링 (LTE)
└── ADB IP 로테이션
```

### Worker 앱 호환성
- Python 3.11+ (버전 체크 필수)
- Windows/Mac 크로스 플랫폼
- AdsPower API URL 설정 가능 (포트 다를 수 있음)
- ADB 경로/디바이스 설정 가능
- setup.py에서 환경 검증 (Python 버전, AdsPower 연결, ADB 연결, 서버 연결)

## 3. 운영자가 하는 일

### 최초 설정 (1회)
1. VPS에 Docker Compose 배포
2. 브랜드 등록 (이름, 상품, 카테고리)
3. 홍보 키워드 설정 (댓글에 녹일 메시지)
4. 타겟 키워드 설정 (영상 검색용)
5. AI 톤 가이드 설정
6. 주간 캠페인 목표 설정
7. 자동 캠페인 켜기
8. 계정 CSV 임포트
9. Worker PC 연결

### 일상 운영 (가끔)
- 대시보드에서 진행 상황 모니터링
- 다이렉트 캠페인 (URL 직접 넣고 즉시 실행)
- 문제 계정 처리
- 프리셋 조정
- 분석 보고 성과 확인

### 운영자가 하지 않는 일 (시스템이 알아서)
- 영상 수집 → 자동
- 캠페인 생성 → 자동 (주간 목표 기반)
- 태스크 분배 → 자동 (Worker에 배정)
- 댓글 생성 → AI 자동
- 좋아요 부스트 → 자동
- 고스트 감지 → 자동
- 계정 보호 (한도, 중복) → 자동
- 에러 알림 → 텔레그램 자동

## 4. 영상 수집

### 수집 전략
```
전략 1: 신규 영상 (4시간마다)
├── 키워드별 YouTube API 검색 (정렬: 최신순)
├── 최신 50개 반환
├── DB에 없는 새 영상만 추가

전략 2: 인기 영상 (1일 1회)
├── 키워드별 YouTube API 검색 (정렬: 조회수순)
├── 상위 50개 반환
├── 조회수 높은 오래된 영상 확보

초기 세팅 (최초 1회):
├── 각 키워드별 조회수순 상위 500개 일괄 수집
└── 이후에는 위 2가지 전략으로 지속
```

### 필터 (단순)
- 댓글 비활성화 영상 제외
- 이미 작업 완료 후 7일 미경과 영상 제외 (같은 브랜드)
- **그 외 필터 없음.** 관련 영상 전부 대상.

### 롱폼/숏폼
- DB에 is_short 표시만
- 작업 순서, 프리셋 선택, 접근 방식 → 차이 없음
- 모든 영상은 검색 or URL로 동일하게 접근

## 5. 캠페인

### 시나리오 캠페인 (자동)
```
운영자: 브랜드 주간 목표 설정 + 자동 캠페인 켜기
시스템:
├── 수집된 영상에서 작업 우선순위에 따라 선택
├── 프리셋 A~J 중 랜덤 배정
├── AI가 댓글 대화 흐름 생성
├── 태스크 분해 → Worker 분배
└── 주간 목표 달성 시 멈춤
```

**운영자가 영상을 수동 선택하지 않음.** 시스템이 알아서 선택.

### 다이렉트 캠페인 (수동)
```
운영자가 직접:
├── URL 여러 개 입력
├── 작업 선택 (댓글/대댓글/좋아요/구독)
├── 댓글: 직접 입력 or 프리셋 선택
├── 대댓글: 직접 입력 or 프리셋 선택
└── 실행
```

### 작업 우선순위
```
1순위: 인기 영상 (조회수 높은 순) — 노출 효과 최대
2순위: 신규 영상 (24시간 이내) — 상단 점령
3순위: 최근 영상 (7일 이내)
4순위: 나머지 미작업 영상
```

### 재작업 규칙
- 같은 계정 + 같은 영상 = 영구 1회만
- 같은 브랜드 + 같은 영상 = 7일 후 재작업 가능
- 다른 브랜드 + 같은 영상 = 제한 없음
- 설정에서 변경 가능

## 6. 댓글 생성

### 3가지 모드
| 모드 | 용도 | 흐름 |
|------|------|------|
| AI 자동 | 시나리오 캠페인 | AI 생성 → 바로 게시 |
| AI + 승인 | 다이렉트 (선택) | AI 생성 → 확인/수정 → 게시 |
| 수동 입력 | 다이렉트 (선택) | 운영자가 직접 입력 |

### 프리셋 시스템
- 기본 프리셋 A~J (시스템 제공, 수정 가능)
- 커스텀 프리셋 추가 가능
- 각 프리셋 = 대화 시나리오 구조 (스텝별 역할, 톤, 좋아요 수, 딜레이 범위)
- 시각적 편집기로 관리 (JSON 아닌 폼 UI)

## 7. 계정 라이프사이클

### 3단계
```
1단계: 계정 생성 (후순위 — 현재는 CSV 임포트)
├── Gmail 가입 + 2FA
├── AdsPower 프로필 생성
└── → 계정 리스트에 등록

2단계: 워밍업 (2~3일)
├── 계정 페이지에서 "워밍업 시작" 클릭
├── Day 1: 시청 + 좋아요 + 채널 설정
├── Day 2: Gmail/검색 + 댓글 + 구독
├── Day 3: 댓글 + 고스트 체크 → 활성 전환
└── 문제 시 → "의심" → 전환 보류

3단계: 작업
├── 자동 캠페인에 투입
├── 일일/주간 한도 자동 관리
├── 고스트 감지 → 쿨다운
└── 폐기 → AdsPower 프로필 삭제
```

### 워커 역할 지정
- 대시보드에서 PC별 역할 설정
- 준비 허용 (로그인/2FA/채널 설정/워밍업)
- 캠페인 허용 (댓글/좋아요/부스트)
- 둘 다 가능

## 8. Worker 실행 방식

### 세션 구조
```
하나의 세션 (한 계정):
├── IP 변경 (ADB 모바일 데이터 토글)
├── AdsPower 프로필 열기 → Playwright CDP 연결
├── YouTube 접속
├── [30%] Gmail 확인
├── [40%] Google 검색 (페르소나 관심사)
├── 자연 브라우징 (숏츠, 추천 영상)
├── 태스크 1 실행 (댓글)
├── 자연 브라우징
├── 태스크 2 실행 (좋아요)
├── ...
├── 세션 제한 (3~8개 태스크 or 20~45분)
└── 프로필 닫기 → 다음 계정

프로필은 1개씩 순차 (IP 분리)
```

### 행동 시뮬레이션
- 마우스 궤적: 베지어 곡선
- 타이핑: 80% 붙여넣기, 20% 타자 (50~200ms/char)
- 간헐적 오타 + 수정 (12%)
- 댓글 전 기존 댓글 읽기 (2~5개, 글자 수 기반 시간)
- 영상 시청: 최대 5분, 화질 안 건드림
- 숏츠: 40% 즉시 넘김, 30% 짧게, 20% 끝까지, 5% 반복
- 알림 확인 5%, 내 채널 방문 3%
- 영상 도달: 70% 검색, 30% 직접 URL

### 자동 로그인
- 계정 정보 (이메일, 비번, TOTP) → 서버 DB에 암호화 저장
- 태스크 실행 시 서버에서 전달 → Worker가 로그인
- 쿠키 만료 시 자동 재로그인
- 캡챠 → 2Captcha 시도 → 실패 시 알림

## 9. 보호 규칙

### 계정 보호
- 일일 댓글 상한: 15개 (설정 가능)
- 일일 좋아요 상한: 50개 (설정 가능)
- 주간 댓글 상한: 70개 (설정 가능)
- 주간 좋아요 상한: 300개 (설정 가능)
- 상한 도달 → 그날 작업 자동 중단

### 영상 보호
- 같은 계정 + 같은 영상 = 1회만
- 같은 브랜드 + 같은 영상 = 7일간 재작업 금지
- 좋아요 중복 방지

### 고스트 감지 3단계
```
1단계: 자연 감지 — 다른 계정이 영상 방문 시 자연스럽게 확인
        최신순 전환 후 확인
2단계: 교차 검증 — 다른 PC + 다른 계정 + 다른 IP (30분 후)
3단계: 24시간 재확인 — 여전히 안 보이면 고스트 확정
        → 계정 쿨다운 + 알림
```

### 실패 처리
```
자동 재시도: IP 실패, 프로필 로딩, 연결 끊김
자동 대체: 캡챠 → 다른 계정, 로그인 실패 → 다른 계정
즉시 중단: 고스트, 계정 정지/차단
알림: 텔레그램 + 대시보드
```

## 10. 에러 로깅

### 기록 항목
모든 에러에 다음을 기록:
- 어떤 계정 (gmail)
- 어떤 영상 (video_id, title)
- 어떤 작업 (task_type)
- 어떤 단계에서 (login, scroll_to_comments, post_comment 등)
- 에러 내용 + 스택 트레이스
- 어떤 Worker PC
- 발생 시각
- 스크린샷 (가능한 경우)

### 대시보드 표시
설정 > 로그 탭에서:
- 심각도별 필터 (critical/warning/info)
- 소스별 필터 (youtube/chrome/ip/system)
- 계정별/영상별 필터
- 스크린샷 보기
- 개별/일괄 해결 처리

### 텔레그램 알림
- 🔴 Worker 연결 끊김
- 🔴 계정 정지/차단
- 🔴 백업 실패
- 🟠 고스트 감지
- 🟠 에러 N회 반복
- 🟢 주간 목표 달성/미달

## 11. 대시보드 페이지 구조

### 사이드바
```
운영:
├── 대시보드 — 전체 현황 한눈에
├── 브랜드 — 브랜드/상품/키워드 관리
├── 타겟 — 수집된 영상 목록 + 수동 추가
├── 캠페인 — 자동 캠페인 현황 + 다이렉트 생성
├── 작업 — 실시간 태스크 큐
└── 분석 — 성과 차트 + 캘린더

관리:
├── 계정 — 계정 관리 + 워밍업 + 문제 계정
├── 워커 — Worker PC 상태 + 관리
└── 설정 — 일반/행동 패턴/프리셋/외관
```

### 대시보드 (홈)
- 5개 통계 카드: 워커 온라인, 진행중 캠페인, 오늘 댓글, 오늘 좋아요, 문제 계정
- 진행중 캠페인 목록 (진행률 바, 어떤 PC 작업중)
- 워커 상태 (온/오프라인)
- 계정 건강 요약 바
- 최근 활동 피드 + 알림

### 브랜드
- 브랜드 카드 목록
- 생성/수정 모달: 이름, 카테고리, 핵심 메시지, 톤 가이드
- **홍보 키워드** (댓글에 녹일 메시지) — 별도 섹션
- **타겟 키워드** (영상 검색용) — 별도 섹션
- 주간 목표 + 자동 캠페인 토글

### 타겟 (영상)
- 수집된 영상 테이블 (제목, 채널, 조회수, 댓글수, 숏/롱, 상태)
- "수동 추가" 버튼 (URL 입력)
- "수집 실행" 버튼 (즉시 수집 트리거)
- **필터 없음** (단순 목록 — 전부 작업 대상)
- 작업 완료 여부 표시

### 캠페인
- 자동 캠페인 목록 (상태, 영상, 브랜드, 프리셋)
- **캠페인 수동 생성 = 다이렉트만** (시나리오는 자동)
- 다이렉트 캠페인 버튼: URL 입력 + 작업 선택 (댓글/대댓글/좋아요/구독)
- 캠페인 행 클릭 → 상세 (스텝, 진행률)

### 작업 (태스크 큐)
- 4개 통계 카드: 대기/진행중/완료/실패
- 태스크 테이블: 유형, 우선순위, 상태, Worker, 진행률
- 컨트롤: 일시정지/취소/우선순위 변경
- 캠페인 단위 일괄 정지/취소

### 분석
- 대시보드 탭: 통계 카드 + 일별 추이 차트 + 브랜드별 비교
- 캘린더 탭: 월간 일별 작업량
- 상세 탭: 브랜드별 성과 테이블

### 계정
- 6개 통계 카드: 전체/활성/워밍업/쿨다운/정지/고스트 (클릭 필터)
- 4개 탭: 전체/워밍업/문제 계정/프로필 풀
- **워밍업 탭에 "워밍업 시작" 버튼** (계정 선택 → Day 선택 → 실행)
- 문제 계정 탭: 일괄 처리 (재검증/쿨다운/폐기)
- 행 클릭 → 상세 시트 (정보, 메트릭, 이력, 상태 변경)

### 워커
- 3개 통계 카드: 온라인, 실행중 태스크, 오늘 완료
- Worker 카드 (PC당 하나): 상태, 프로필 수, 태스크, 역할 뱃지
- "워커 추가" 버튼 (이름 + 시크릿 → 토큰 발급)
- **일시정지 버튼** (태스크 배정 중단)
- 카드 클릭 → 상세 (태스크 목록, 에러 로그)

### 설정
