# HYDRA v2 — 계정/브라우저 자동화 스펙

> 서브시스템 4/5: 계정/브라우저 자동화
> 작성일: 2026-04-16
> 상태: 승인됨

## 1. 개요

기존 계정 라이프사이클, AdsPower 연동, 워밍업 방식을 유지하되 멀티 PC 환경에 적응.

## 2. 유지 (기존 방식)

- 계정 라이프사이클: registered → warmup → active → cooldown → retired
- Gmail 자동 가입 + TOTP 2FA (후속 사이클에서 자동화 강화)
- 페르소나 생성 (슬롯 기반)
- 워밍업 그룹 A~E (기간별 분류)

## 3. 변경 (멀티 PC 적응)

### 3.1 AdsPower 연동

- 각 Worker가 자기 PC의 AdsPower 로컬 API 호출
- 서버는 "어떤 프로필 열어" 명령만 → Worker가 실행
- 프로필 잠금은 서버에서 관리
- 같은 AdsPower 계정으로 각 PC에 로그인 → 프로필 공유

### 3.2 계정 생성 위치

- 서버가 "계정 생성" 태스크 발행 → Worker가 실행
- 생성된 계정 정보 → 서버 DB에 등록
- 어떤 PC에서든 계정 생성 가능

### 3.3 워밍업 분배

- 서버가 워밍업 태스크를 Worker들에 분배
- 여러 PC에서 동시 워밍업 가능
- 프로필 잠금으로 중복 방지

## 4. 추가 기능

### 4.1 계정-프로필 매칭

- 대시보드에서 계정 등록 시 AdsPower 프로필 ID 매칭
- 한번 매칭되면 고정 (수동 해제만 가능)
- 프로필 없는 계정 → 자동 생성 or 풀에서 배정

### 4.2 계정 대량 등록

- CSV 임포트 (이메일, 비번, 프로필 ID 등)
- 임포트 시 기존 이력과 충돌 검증

### 4.3 계정 상태 자동 전환

- 워밍업 기간 완료 → 자동 활성 전환
- 일일 상한 도달 → 자동 휴식
- 고스트 감지 → 자동 쿨다운
- 쿨다운 기간 완료 → 테스트 댓글 → 복귀 or 폐기
- 폐기 시 AdsPower 프로필 자동 삭제

### 4.4 아바타 관리

- 서버에 아바타 업로드
- 계정 등록 시 자동/수동 배정
- Worker가 워밍업 시 서버에서 다운로드 → 유튜브 채널 설정

## 5. 계정 자동 생성 (후속 사이클)

1차 구현에서는 계정 페이지에 [계정 생성] 탭 UI만 만들어놓고 "준비중" 표시. CSV 임포트로 당분간 대체. 후속 사이클에서 Gmail 자동 가입 + AdsPower 프로필 자동 생성 + 페르소나 자동 배정 구현.

## 6. 데이터 마이그레이션

기존 20개 계정 + 모든 작업 이력 완벽 보존. SQLite → PostgreSQL 이전 시 데이터 무결성 검증 필수.
