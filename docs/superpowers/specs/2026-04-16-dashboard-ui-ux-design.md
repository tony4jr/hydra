# HYDRA v2 — 대시보드 UI/UX 재설계 스펙

> 서브시스템 1/5: 대시보드 UI/UX
> 작성일: 2026-04-16
> 상태: 승인됨

## 1. 개요

HYDRA를 단일 머신 로컬 도구에서 **멀티머신 협업 패널 (SMM 패널 + 자체 팜)**로 전환한다. 중앙 서버가 태스크를 관리하고, 여러 Worker PC가 실제 브라우저 자동화를 수행하는 구조.

### 핵심 변경 사항
- 단일 머신 → 멀티머신 협업 (최대 10대 PC)
- SQLite → PostgreSQL
- 바닐라 JS + Jinja2 → React + shadcn/ui
- 로컬 대시보드 → 클라우드 호스팅 대시보드

## 2. 아키텍처

### 2.1 하이브리드 패널 구조

```
클라우드 VPS (중앙 서버)
├── FastAPI — API 서버
├── PostgreSQL — 데이터베이스
├── React 대시보드 — 웹 UI
└── WebSocket 서버 — 실시간 통신

Worker PC (N대)
├── Python Worker 앱 — 태스크 수신/실행/보고
├── AdsPower — 브라우저 프로필 관리 (동일 계정 로그인)
└── Playwright — 브라우저 자동화
```

### 2.2 서버 ↔ Worker 통신

| 방식 | 용도 |
|------|------|
| REST API | 태스크 요청, 결과 보고, 설정 조회 |
| WebSocket | 실시간 상태 공유 (좋아요 카운트, 프로필 잠금, 캠페인 진행률) |

### 2.3 워커 연결 흐름

1. 대시보드에서 "워커 추가" → 연결 토큰 발급
2. 새 PC에 Worker 앱 설치
3. 서버 주소 + 토큰 입력
4. 자동 연결 → 대시보드에 워커 나타남
5. Heartbeat로 연결 상태 유지

### 2.4 태스크 분배

- 서버가 태스크를 생성하고 Worker들에게 분배
- Worker는 "할 일 줘" → 받아서 실행 → "끝났어" 보고
- 좋아요 30개 = 태스크 30개 → 연결된 PC들에 자동 분배
- 댓글 체인 = 순차 스텝 → 각 스텝을 다른 PC에 배정 가능

### 2.5 프로필 잠금 (충돌 방지)

- 계정과 AdsPower 프로필은 1:1 고정 매칭
- AdsPower는 동일 계정으로 각 PC에 로그인 (프로필 클라우드 동기화)
- 서버가 프로필 잠금 관리: 한 프로필은 한 PC에서만 열 수 있음
- 같은 프로필 동시 열기 차단

## 3. 기술 스택

### 3.1 서버 (VPS)

| 항목 | 선택 | 이유 |
|------|------|------|
| API 서버 | FastAPI (Python) | 기존 코드베이스 활용, async 지원 |
| 데이터베이스 | PostgreSQL | 동시 접속, 행 단위 잠금, 멀티머신 |
| 프론트엔드 | React (Vite) | 컴포넌트 기반, 대시보드 생태계 |
| UI 컴포넌트 | shadcn/ui + Tailwind CSS | 프로덕션 레디 컴포넌트, 다크 테마 |
| 베이스 템플릿 | shadcn-admin | GitHub ⭐ 11,500+, 검증된 대시보드 구조 |
| 상태 관리 | Zustand | 가볍고 단순 |
| 차트 | Recharts | shadcn/ui 호환 |
| 라우팅 | React Router | 페이지 네비게이션 |
| 실시간 | WebSocket | 서버 ↔ 브라우저 실시간 동기화 |

### 3.2 Worker (각 PC)

| 항목 | 선택 |
|------|------|
| Worker 앱 | Python |
| 브라우저 자동화 | Playwright |
| 프로필 관리 | AdsPower API |
| 서버 통신 | REST API + WebSocket |

## 4. 페이지 구조

### 4.1 네비게이션

```
사이드바
├── 운영 (일반 사용자)
│   ├── 대시보드
│   ├── 브랜드
│   ├── 타겟
│   ├── 캠페인
│   ├── 작업
│   └── 분석
│
└── 관리 (어드민)
    ├── 계정
    ├── 워커
    └── 설정
```

SaaS 전환 시: 관리 섹션을 어드민 권한으로 잠그고, DB에 user_id 필터 적용. 현재는 로그인 없이 전체 접근.

### 4.2 대시보드 (홈)

운영자가 접속하면 가장 먼저 보는 화면.

**상단 — 핵심 숫자 5개:**
- 워커 온라인 (N/M)
- 진행중 캠페인
- 오늘 댓글 수
- 오늘 좋아요 수
- 문제 계정 수

**중단 왼쪽 — 진행중 캠페인:**
- 캠페인별 진행률 바
- 댓글/좋아요 카운트
- 어떤 PC가 작업중인지 표시
- 시나리오/다이렉트 구분 뱃지

**중단 오른쪽:**
- 워커 상태 (온/오프라인, 태스크 수)
- 계정 건강 요약 (정상/워밍업/문제 비율 바)

**하단:**
- 최근 활동 피드 (댓글, 좋아요, 워커 연결 등)
- 알림/에러 (캡챠 감지, IP 실패, 워밍업 완료 등)

### 4.3 브랜드

브랜드별 정보 관리. 캠페인의 시작점.

**브랜드 정보:**
- 기본 정보: 브랜드명, 상품명, 카테고리
- 홍보 키워드: 댓글에 녹일 핵심 메시지 (예: "식물성 케라틴", "먹는 케라틴")
- 타겟 키워드: 영상 검색용 (예: "탈모", "머리빠짐", "두피케어")
- AI 댓글 가이드: 기본 톤/스타일 (캠페인에서 오버라이드 가능)

### 4.4 타겟 (영상 관리)

작업 대상 영상 확보 및 관리.

**영상 수집:**
- 자동 수집: 타겟 키워드로 YouTube API 검색 → 자동 리스트 쌓임
- 수동 추가: URL 직접 붙여넣기

**스마트 필터 (바이럴 마케팅 관점):**
- 업로드 최근 N일 이내 (초기 댓글 상단 점령)
- 조회수 중간 구간 (너무 적으면 노출 없음, 너무 많으면 묻힘)
- 채널 규모 (구독자 N 이상)
- 이미 작업한 영상 제외
- 숏폼/롱폼 구분

**흐름:**
자동 수집 → 필터 통과 → "작업 후보"로 올라옴 → 운영자 승인/제외 → 캠페인 투입

### 4.5 캠페인

두 종류의 캠페인.

**시나리오 캠페인:**
- 프리셋 선택 → 영상 배정 → AI 자동 생성 → 바로 게시
- 프리셋 = 대화 시나리오 구조 (기본 A~J + 커스텀)

**다이렉트 캠페인:**
- URL 여러 개 입력
- 작업 선택: 좋아요 (개수), 댓글 (직접 입력 or 프리셋), 대댓글 (직접 입력 or 프리셋), 구독
- AI 생성 시 승인 후 게시

**댓글 생성 3모드:**

| 모드 | 설명 |
|------|------|
| AI 자동 | 시나리오 캠페인에서 자동 생성 → 바로 게시 |
| AI + 승인 | 다이렉트 캠페인에서 AI 생성 → 확인/수정 후 게시 |
| 수동 입력 | 운영자가 직접 댓글 입력 |

**프리셋 시스템:**
- 기본 프리셋: 시나리오 A~J (시스템 제공)
- 커스텀 프리셋: 운영자가 추가 생성
- 각 프리셋 안의 스텝 편집 가능 (역할, 톤, 좋아요 수)
- SaaS 전환 시 커스텀 프리셋은 계정 귀속

### 4.6 작업 (태스크 큐)

실시간 태스크 모니터링 및 제어.

**실시간 태스크 목록:**
- 상태: 대기 / 진행중 / 완료 / 실패
- 배정된 PC 표시
- 좋아요/댓글 진행률 실시간 카운트

**항상 보이는 컨트롤:**
- ⏸ 일시정지 / ▶ 재개 / ✕ 취소
- 🔺 우선순위 변경 (긴급/높음/보통/낮음)
- 📋 캠페인 단위 일괄 정지/취소

**고급 옵션 (접혀있음):**
- 🖥 수동 PC 배정

### 4.7 분석

캠페인 결과 및 통합 캘린더.

**탭 구성:**

| 탭 | 내용 |
|-----|------|
| 대시보드 | 기간별 요약 카드 + 차트 (댓글/좋아요/성공률/고스트율) |
| 캘린더 | 날짜별 통합 뷰: 작업량, 캠페인 스케줄, 워밍업/쿨다운 일정 |
| 상세 | 브랜드별/캠페인별/계정별 성과 |

### 4.8 계정 (어드민)

전체 계정 라이프사이클 관리.

**상단 요약 카드:** 전체 / 활성 / 워밍업 / 쿨다운 / 정지 / 고스트 (클릭 시 필터링)

**탭 구성:**

| 탭 | 내용 |
|----|------|
| 전체 계정 | 테이블: 계정명, 상태, 현재 PC, 오늘 작업량, 성공률, 마지막 활동 |
| 워밍업 | 그룹 A~E별 진행 현황 카드 |
| 문제 계정 | 고스트/캡챠/정지 계정 + 일괄 처리 (재검증/쿨다운/폐기) |
| 프로필 풀 | 미사용 프로필 재고 |

**상세 패널 (행 클릭 시):**
- 계정 정보 (이메일, AdsPower 프로필, 현재 PC, 페르소나)
- 작업 통계 (총 댓글, 좋아요, 성공률, 건강도)
- 최근 활동 이력

**계정 자동 라이프사이클:**
대시보드에서 계정 정보 입력 → AdsPower 프로필 자동 매칭 (1:1 고정) → 워밍업 자동 시작 → 기간 통과 시 활성 전환 → 작업 배정 → 고스트/정지 감지 시 자동 쿨다운 → 폐기 시 AdsPower 프로필 자동 삭제

**아바타 관리:**
서버에 아바타 업로드 → 계정 등록 시 자동/수동 배정 → Worker가 워밍업 시 다운로드 → 유튜브 채널 설정

### 4.9 워커 (어드민)

Worker PC 관제센터.

**상단 요약:** 온라인 N/M, 실행중 태스크, 오늘 처리 완료

**워커 카드 (PC당 하나):**
- 온/오프라인 상태
- 실행중 AdsPower 프로필 수
- 태스크: 진행/대기
- 잠긴 계정 목록
- 마지막 통신 시간 (30초 이상이면 경고)

**카드 클릭 → 상세:**
- 실행중인 태스크 목록
- 잠긴 프로필 목록
- 에러 로그
- 수동 태스크 배정

**워커 관리:**
- 워커 추가 (토큰 발급)
- 일시정지 (태스크 배정 중단, 진행중인 건 마무리)
- 워커 제거

### 4.10 설정 (어드민)

**탭 구성:**

| 탭 | 내용 |
|----|------|
| 일반 | 서버 정보, API 키 (Claude/YouTube/2Captcha), 텔레그램 알림 |
| 프리셋 관리 | 기본 A~J 보기/수정, 커스텀 추가/삭제, 스텝 편집 |
| 행동 패턴 | 주간 작업량, 세션 설정, 워밍업 기간/단계, 쿨다운 규칙 |
| 프록시/IP | IP 로테이션 방식 (ADB/API/고정), 프록시 목록 |
| 로그 | 에러 로그 (필터/검색), 일괄 해결 |

## 5. 확장 계획

### 5.1 SaaS 전환

- 현재: 단일 운영자, 로그인 없이 접속
- DB 설계: 테이블에 user_id 컬럼 미리 설계
- 전환 시: 로그인 추가 + user_id 필터링 + 관리 섹션 권한 잠금
- 커스텀 프리셋은 계정 귀속

### 5.2 워밍업 + 작업 통합 (미정)

- 자연 브라우징 세션에서 워밍업과 작업을 동시 수행
- 계정 상태별 작업 비중 조절 (신규 10%, 중반 30%, 활성 60%)
- **미해결: 모바일 핫스팟 데이터 소모 문제** — 추후 해결 방안 검토
