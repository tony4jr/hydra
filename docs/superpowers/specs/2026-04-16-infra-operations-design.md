## 1. 개요

VPS 서버 배포, Worker PC 배포, IP 로테이션, 모니터링, 백업, 자동 업데이트를 다루는 운영 인프라.

## 2. VPS 서버 배포

### 2.1 Docker Compose

```
docker-compose.yml
├── fastapi — API 서버 (REST + WebSocket)
├── postgres — PostgreSQL 데이터베이스
├── nginx — React 빌드 서빙 + 리버스 프록시
└── docker-compose up -d 한 줄이면 끝
```

### 2.2 설치 흐름

1. VPS 생성 (월 5천~1만원)
2. 코드 클론 + .env 설정 (API 키, DB 비밀번호 등)
3. `docker-compose up -d`
4. 브라우저로 대시보드 접속

### 2.3 서버 업데이트

```
git pull → docker-compose restart
DB 마이그레이션 필요 시 자동 실행
```

## 3. Worker PC 배포

### 3.1 설치 흐름

1. 설치 스크립트 다운로드 + 실행
2. 서버 주소 + 연결 토큰 입력
3. AdsPower 로그인
4. 자동 연결 → 대시보드에 워커 나타남

Windows/Mac 모두 지원.

### 3.2 Worker 자동 업데이트

브라우저 자동화 코드 변경 시, 운영자가 서버에만 배포하면 Worker가 알아서 업데이트:

1. Worker가 주기적으로 서버에 버전 체크 (5분마다)
2. 새 버전 감지 → 진행중 태스크 마무리
3. 새 코드 다운로드 → 자동 교체 → 재시작
4. 대시보드에 업데이트 완료 표시

운영자가 PC별로 돌아다니며 업데이트할 필요 없음.

## 4. 네트워크 / IP 로테이션

### 4.1 기본 방식

모든 Worker PC는 **휴대폰 USB 테더링**으로 인터넷 연결. IP 로테이션은 ADB를 통한 모바일 데이터 토글.

```
Worker PC ← USB 테더링 → 휴대폰 (모바일 데이터)
IP 변경: ADB로 모바일 데이터 off → 12초 대기 → on → 새 IP 할당
```

### 4.2 추가 옵션 (설정 탭에서 활용 가능)

| 방식 | 설명 | 상태 |
|------|------|------|
| ADB 모바일 데이터 토글 | 기본, 모든 PC | 활성 |
| 프록시 API (회전형) | 필요 시 사용 | 탭만 준비 |
| 고정 프록시 | 필요 시 사용 | 탭만 준비 |

PC별로 IP 방식을 다르게 설정 가능. 서버는 "IP 변경해" 명령만 → Worker가 자기 설정대로 실행.

## 5. 모니터링

### 5.1 실시간 대시보드

- Worker 헬스체크 (heartbeat 30초)
- 태스크 성공/실패율
- 캠페인 진행률
- 계정별 상태
- 프로필 잠금 현황

### 5.2 텔레그램 알림

| 긴급도 | 알림 |
|--------|------|
| 🔴 긴급 | Worker 연결 끊김, 계정 정지/차단, 백업 실패 |
| 🟠 경고 | 고스트 감지, 에러 N회 반복 |
| 🟢 정보 | 주간 목표 달성/미달 |

대시보드 + 텔레그램 동시 발송.

### 5.3 로그 (중앙 집중)

- 모든 로그 서버 DB에 저장
- Worker 로그도 서버에 전송
- 설정 페이지 > 로그 탭에서 검색/필터
- 일괄 해결 처리

## 6. 백업

- N일간 보관 (설정 가능)
- 백업 실패 시 텔레그램 알림
- VPS 이전/복구 시 백업에서 복원
