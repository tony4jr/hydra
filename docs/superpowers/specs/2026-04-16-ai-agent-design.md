# HYDRA v2 — AI 에이전트 시스템 스펙

> 서브시스템 3/5: AI 에이전트
> 작성일: 2026-04-16
> 상태: 승인됨

## 1. 개요

기존 5개 에이전트를 3개로 통합. 프리셋 기반 대화 흐름 생성에 최적화. AI 생성은 서버에서 수행, Worker는 텍스트 붙여넣기만.

## 2. 에이전트 구조

### 2.1 변경 사항

| 기존 | 변경 |
|------|------|
| comment_agent (프로모 댓글) | content_agent로 통합 |
| reply_agent (대댓글) | content_agent로 통합 |
| casual_agent (비프로모) | content_agent로 통합 |
| persona_agent | 유지 |
| keyword_agent | 유지 |

### 2.2 content_agent

프리셋 전체를 하나의 대화 흐름으로 생성. 맥락이 이어져서 자연스러움.

**입력:**
- 브랜드 정보 (상품, 홍보 키워드, AI 가이드)
- 프리셋 구조 (스텝별 역할, 톤)
- 영상 정보 (제목, 설명)
- 페르소나 (각 계정의 캐릭터)
- 톤 오버라이드 (있으면)
- 이전 생성 결과 (중복 방지용)

**출력:**
- 스텝별 텍스트 배열

### 2.3 모델 배정

| 용도 | 모델 | 이유 |
|------|------|------|
| 프로모 댓글 (브랜드 메시지 포함) | Sonnet | 품질 중요, 브랜드 가이드 준수 |
| 비프로모 / 동조 대댓글 | Haiku | 가벼운 작업, 비용 절감 |
| 페르소나 생성 | Haiku | 한번 만들면 끝 |
| 키워드 확장 | Haiku | 단순 작업 |

## 3. 하네스 구조

```
content_agent 하네스
├── 입력 검증
│   └── 브랜드, 프리셋, 영상 정보 완전한지 체크
├── 프롬프트 조립
│   ├── 시스템 프롬프트 (역할 정의 + 금지 규칙)
│   ├── 브랜드 컨텍스트
│   ├── 프리셋 구조 (스텝별 역할, 톤 지시)
│   ├── 영상 컨텍스트
│   ├── 페르소나
│   └── 중복 방지 (이전 생성 결과 참고)
├── 모델 분기
│   ├── 프로모 스텝 → Sonnet
│   └── 동조/캐주얼 스텝 → Haiku
├── 생성 + 검증 루프 (최대 3회)
│   ├── 금지어 포함? → 재생성
│   ├── 너무 광고스러움? → 재생성
│   ├── 이전 결과와 너무 비슷? → 재생성
│   ├── 길이 적절? → 범위 밖이면 재생성
│   └── 스텝 수 맞는지? → 누락 시 재생성
└── 출력: 스텝별 텍스트 배열
```

## 4. 품질 보장

- 같은 프리셋 반복 사용 시 매번 다른 표현 생성
- 브랜드 금지어 자동 필터링
- 너무 광고스러운 표현 감지 → 재생성
- 댓글 길이 자연스럽게 변동
- 맞춤법 살짝 틀리기 (선택, 사람스러움)

## 5. AI 생성 위치

서버에서 생성 (Worker가 아님):
- 캠페인 생성 시 → 서버에서 AI로 댓글 텍스트 미리 생성
- 생성된 텍스트를 태스크에 포함
- Worker는 텍스트 받아서 붙여넣기만
- API 키 관리 중앙화

## 6. 기존 데이터 호환

기존 작업 이력과 절대 꼬이면 안 됨. 이전 댓글 내역을 중복 방지 참고 데이터로 활용.
