# HYDRA v2 — 코어 엔진 아키텍처 스펙

> 서브시스템 2/5: 코어 엔진
> 작성일: 2026-04-16
> 상태: 승인됨

## 1. 개요

캠페인 실행, 태스크 분배, 스케줄링, 보호 규칙, 실패 처리, 고스트 감지를 다루는 비즈니스 로직 핵심.

## 2. 캠페인 실행

### 2.1 캠페인 종류

| 종류 | 설명 | 댓글 생성 |
|------|------|----------|
| 시나리오 캠페인 | 프리셋 기반, 브랜드 주간 목표에 연결 | AI 자동 → 바로 게시 |
| 다이렉트 캠페인 | URL 여러 개 입력, 작업 선택 | AI+승인 or 수동 입력 |

### 2.2 캠페인 생성 시 설정 항목

- 브랜드 선택
- 프리셋 선택 (시나리오 A~J or 커스텀)
- 타겟 영상 선택 (승인된 후보 영상)
- 실행 모드 (자동 분산 / 긴급)
- AI 톤 오버라이드 (선택, 안 하면 브랜드 기본값)

### 2.3 주간 목표 기반 자동 캠페인 생성

```
브랜드에 주간 목표 설정 (예: 15개/주)
  → 타겟 영상 API 자동 수집 (상시 가동) + 수동 추가
  → 스마트 필터 통과 → 후보 쌓임
  → 시스템이 후보에서 골라 캠페인 자동 생성
  → 프리셋 랜덤 배정 (or 운영자 설정 비율)
  → 자동 분산 실행
  → 주간 목표 채워지면 멈춤
  → 다음 주 반복
```

후보 영상 부족 시 대시보드 알림.

## 3. 타이밍 및 행동 패턴

### 3.1 핵심 원칙

**모든 타이밍 = 범위(min~max) + 가우시안 랜덤 분포. 고정값 절대 금지.**

자동화 도구처럼 보여서는 안 되며, 모든 행동은 사람처럼 보여야 함. 질보다 양, 오래가는 것이 최우선.

### 3.2 기본 타이밍 범위

| 액션 | 간격 | 비고 |
|------|------|------|
| 좋아요 | 15~90초 | 랜덤 |
| 메인 댓글 | 3~15분 | 랜덤 |
| 대댓글 | 5~30분 | 메인 댓글 이후 |
| 영상 이동 | 15~60분 | 랜덤 |
| 좋아요 부스트 시작 | 2~8분 후 | 댓글 작성 이후 |

모든 기본값은 설정 > 행동 패턴에서 조정 가능. 프리셋 안에도 스텝별 타이밍 범위 포함.

### 3.3 실행 모드

| 모드 | 설명 |
|------|------|
| 자동 분산 | 기본. 위 타이밍 범위로 자연스럽게 분산 |
| 긴급 | 간격 최소화. 영상이 지금 터지고 있을 때 |

## 4. 보호 규칙

### 4.1 영상 보호

| 규칙 | 설명 |
|------|------|
| 영상당 캠페인 수 제한 | N일 이내 최대 M개 캠페인 (설정 가능) |
| 같은 계정 중복 방지 | 한 계정이 같은 영상에 중복 댓글 불가 |
| 좋아요 중복 방지 | 이미 좋아요 누른 계정은 자동 건너뜀 |

### 4.2 계정 보호

| 규칙 | 설명 |
|------|------|
| 일일 댓글 상한 | N개/일 (설정 가능) |
| 일일 좋아요 상한 | N개/일 (설정 가능) |
| 일일 총 활동 상한 | N개/일 (설정 가능) |
| 주간 상한 | N개/주 (설정 가능) |
| 상한 도달 시 | 자동으로 그날 작업 중단, 다음 날 재개 |

## 5. 태스크 분배 (멀티 PC)

### 5.1 분배 방식

- 서버가 캠페인 → 개별 태스크로 분해
- Worker가 "할 일 줘" → 서버가 배정 → 실행 → 결과 보고
- 좋아요 30개 = 태스크 30개 → 연결된 PC들에 자동 분배
- 댓글 체인 = 순차 스텝 → 각 스텝을 다른 PC에 배정 가능
- 프로필 잠금으로 동시 접속 방지

### 5.2 우선순위

4단계: 긴급 > 높음 > 보통 > 낮음. 작업 페이지에서 변경 가능.

## 6. 실패 처리

### 6.1 자동 재시도

| 실패 유형 | 처리 |
|----------|------|
| IP 변경 실패 | 재시도 3회 → 성공 시 계속 |
| 프로필 로딩 실패 | 재시도 2회 → 실패 시 다른 PC에 재배정 |
| Worker 연결 끊김 | 해당 태스크 회수 → 다른 PC에 재배정 |
| 유튜브 일시 오류 | 5분 후 재시도 |

### 6.2 자동 대체

| 실패 유형 | 처리 |
|----------|------|
| 캡챠 감지 | 해당 계정 일시 정지 → 다른 계정으로 재배정 |
| 로그인 실패 | 해당 계정 "문제" 처리 → 다른 계정 투입 |
| 재시도 상한 초과 | 다른 계정/PC 조합으로 자동 전환 |

### 6.3 즉시 중단 + 알림

| 실패 유형 | 처리 |
|----------|------|
| 고스트 감지 | 해당 계정 즉시 작업 중단 → 알림 |
| 계정 정지/차단 | 즉시 중단 → 알림 |
| 같은 에러 N회 반복 | 시스템 이상 → 텔레그램 알림 |

## 7. 고스트 감지

### 7.1 3단계 검증

**1단계: 자연 감지**
- 다른 계정이 해당 영상 방문할 때 자연스럽게 확인
- 별도 감지 작업을 돌리지 않음 — 일반 작업 흐름 안에서 확인
- 반드시 "최신순" 정렬로 전환 후 확인
- 댓글 수 많은 영상 (1000개+)은 최신순에서만 판단

**2단계: 교차 검증 (의심 시에만)**
- 다른 PC + 다른 계정 + 다른 IP로 확인
- 최소 2개 다른 조합에서 안 보이면 → 고스트 확정
- 1개에서만 안 보이면 → 보류
- 시간 간격 두고 확인 (바로 하면 부자연스러움)

**3단계: 판정 + 처리**
- 고스트 확정 → 해당 계정 즉시 작업 중단 + 알림
- 보류 → 24시간 후 재확인 → 여전히 안 보이면 확정
- 고스트 확정 계정: 자동 쿨다운 → 복귀 시 테스트 댓글 1개로 검증 → 실패 시 폐기

### 7.2 노출 판정 기준

| 인기순 | 최신순 | 판정 |
|--------|--------|------|
| O | O | 정상 |
| X | O | 정상 (묻힌 것) |
| X | X | 의심 → 교차 검증 |

### 7.3 상단 노출 추적 (데이터 수집)

캠페인별로 댓글 위치(상단/중단/하단/안보임) 추적. 프리셋별, 좋아요 수별, 타이밍별 상단 노출 성공률을 분석 페이지에서 비교. 이 데이터를 축적하여 YouTube 댓글 알고리즘 최적 전략 도출.
