# HYDRA v2 — 브라우저 자동화 엔진 스펙

> 작성일: 2026-04-17
> 상태: 승인됨

## 1. 개요

Worker PC에서 실제 브라우저를 조작하여 YouTube 댓글/좋아요/워밍업 등을 수행하는 자동화 엔진. AdsPower로 프로필을 열고, Playwright CDP로 조작한다. 기존 행동 로직(actions.py, behavior.py)을 최대한 활용하되, 오케스트레이션을 Worker 태스크 기반으로 재설계한다.

### 핵심 원칙
- **모든 행동은 사람처럼.** 고정값/패턴 절대 금지, 가우시안 랜덤.
- **질보다 양, 오래가기.** 공격적보다 안전하게.
- **화질 건드리지 않음.** LTE로 운영, 시청 시간 관리로 데이터 절약.

## 2. 아키텍처

### 2.1 역할 분리

```
AdsPower:
├── 프로필 관리 (지문, 쿠키, 프록시)
├── 브라우저 열기 → WebSocket 디버거 URL 반환
└── 브라우저 닫기

Playwright:
├── CDP로 AdsPower 브라우저에 연결
├── 페이지 조작 (클릭, 타이핑, 스크롤)
├── 요소 대기, 동적 로딩 처리
└── 에러 감지, 스크린샷

IP 로테이션:
├── ADB 모바일 데이터 토글 (LTE)
├── 프로필 열기 전에 IP 변경
└── 1 IP = 1 계정 (동시 사용 금지)
```

### 2.2 세션 구조

모드 분리 없이 하나의 세션에서 자연스럽게 섞어서 실행:

```
Worker가 Account A 세션 시작:
├── IP 변경 (ADB)
├── AdsPower 프로필 열기 → WebSocket URL
├── Playwright CDP 연결
├── YouTube 접속
│
├── [자연 브라우징] 숏츠 2~3개 시청
├── [30%] Gmail 확인
├── [40%] Google 검색 (페르소나 관심사)
├── [태스크 1] 영상 X → 댓글 작성
├── [자연 브라우징] 추천 영상 시청
├── [태스크 2] 영상 Y → 좋아요 부스트
├── [5%] 알림 확인
├── [태스크 3] 영상 Z → 대댓글
│
├── 세션 제한 도달 (최대 N개 태스크 or M분)
├── Playwright 연결 해제
└── AdsPower 프로필 닫기 → 다음 계정
```

프로필은 **1개씩 순차 실행** (IP 분리 위해 동시 열기 안 함).

## 3. 계정 라이프사이클 3단계

### 3.1 계정 생성 모드 (후순위)

```
Gmail 가입 + 2FA 설정 + AdsPower 프로필 생성
  → 초기 워밍업 (며칠)
  → 완료 시 계정 리스트에 등록
  → (별도 설계 — 이 스펙에서는 다루지 않음)
```

### 3.2 워밍업 모드 (지금 구현)

기존 20개 계정을 계정 생성 완료 상태로 간주하고, 워밍업 실행.

**워밍업 기간: 2~3일, 점진적 활동 증가.**

**Day 1 — 가볍게 시작 (계정당 15~25분):**
- YouTube 접속
- 채널 설정 (이름, 아바타) — 첫 실행 시 1회
- 숏츠 시청 (랜덤 패턴)
- 영상 1~2개 시청 (최대 1~3분)
- 좋아요 1~2개

**Day 2 — 활동 늘리기 (계정당 20~35분):**
- Gmail 확인 (30%)
- Google 검색 1회 (40%)
- 영상 2~3개 시청
- 좋아요 2~3개
- 비프로모 댓글 1~2개
- 구독 1개

**Day 3 — 활성 전환 준비 (계정당 20~35분):**
- 영상 2~3개 시청
- 댓글 3~5개
- 좋아요 여러 개
- **Day 2에 남긴 댓글 고스트 체크** (영상 재방문 → 최신순 → 확인)
- 통과 시 "활성" 전환, 안 보이면 "의심" → 전환 보류

### 3.3 작업 모드

프리셋 기반 캠페인 작업. 활성 계정만 투입.
- 댓글/대댓글 (프리셋 스텝 기반)
- 좋아요 부스트
- 다이렉트 캠페인
- 고스트 체크 (3단계)

### 3.4 워커 역할 지정

대시보드 워커 설정에서 PC별 역할 설정:

```
PC-1: [✅ 준비 작업] [✅ 캠페인 작업]  ← 운영자 가까이
PC-2: [❌ 준비 작업] [✅ 캠페인 작업]  ← 원격
PC-3: [❌ 준비 작업] [✅ 캠페인 작업]  ← 원격
```

서버가 태스크 배정 시 워커 역할을 확인하고 필터링.

## 4. 행동 시뮬레이션 (사람처럼)

### 4.1 기존 로직 활용 (actions.py)

| 함수 | 동작 | 유지 |
|------|------|------|
| type_human() | 80% 붙여넣기, 20% 타자 (50~200ms/char) | ✅ |
| watch_video() | 시청 중 20% 댓글 스크롤, 10% 설명 펼치기, 5% 일시정지 | ✅ |
| handle_ad() | 60% 스킵, 25% 시청, 15% 뒤로가기 | ✅ |
| post_comment() | 생각 1~3초 → 입력 → 재읽기 1~3초 → 게시 | ✅ |
| post_reply() | 대댓글 클릭 → 입력 → 게시 | ✅ |
| scroll_to_comments() | 400~700px × 5회, 1.5~3초 간격 | ✅ |
| click_like_button() | 영상/댓글 좋아요 | ✅ |
| check_ghost() | DOM 기반 댓글 존재 확인 | ✅ |

### 4.2 기존 로직 활용 (behavior.py)

| 함수 | 동작 | 유지 |
|------|------|------|
| plan_daily() | 가우시안 분포 일일 목표 | ✅ |
| pick_action() | 행동 가중치 랜덤 선택 | ✅ |
| pick_watch_duration() | 시청 시간 분포 | ✅ (수정) |
| is_natural_activity_hour() | KST 07:00~01:00 | ✅ |

### 4.3 추가 행동

**영상 도달 방식:**
- 70% 검색/추천으로 자연스럽게 도달
- 30% 직접 URL (폴백 또는 설정)
- 검색 시 N초 이내 못 찾으면 직접 URL로 전환
- 비율 설정 가능

**댓글 전 읽기 행동:**
- 기존 댓글 2~5개 영역 스크롤 + 중간중간 멈춤
- 읽는 시간 랜덤 (3~10초)
- 그 다음 댓글 작성

**마우스 궤적 시뮬레이션:**
- 클릭 전 마우스가 목표 지점으로 자연스럽게 이동
- 직선이 아닌 곡선 궤적
- 이동 속도 랜덤

**간헐적 오타 + 수정:**
- 10~15% 확률로 오타 발생
- 백스페이스로 수정 후 올바르게 재입력
- 타자 입력 시에만 (붙여넣기 시엔 안 함)

**알림/프로필 확인:**
- 5% 확률로 알림 벨 클릭
- 3% 확률로 내 채널 방문
- 세션 중 최대 1회

### 4.4 숏츠 시청 패턴

```
스와이프 후 행동:
├── 즉시 넘김 (1~2초 보고 패스): 40%
├── 짧게 시청 (3~10초): 30%
├── 끝까지 시청 (15~60초): 20%
├── 반복 시청 (한번 더): 5%
├── 좋아요 (시청한 것 중): 10%
├── 댓글 열어보기: 5%
└── 총 숏츠 수: 2~15개 (랜덤)
```

### 4.5 영상 시청 패턴

```
시청 시간 (영상 길이별 최대):
├── 숏츠 (~60초): 그대로
├── 짧은 영상 (1~5분): 최대 3분
├── 중간 영상 (5~15분): 최대 5분
├── 긴 영상 (15분+): 최대 3~5분
└── 절대 5분 초과 안 함

시청 행동:
├── 썸네일 클릭 → 3초 안에 뒤로가기 (관심 없음): 15%
├── 30초~1분 보다 이탈: 25%
├── 중간까지 시청: 30%
├── 시간 바 앞으로 건너뛰기 → 좀 보다 이탈: 20%
├── 댓글부터 스크롤 (영상 거의 안 봄): 10%
└── 화질: 건드리지 않음 (auto)

시청 중 랜덤 행동:
├── 댓글 영역 스크롤 (읽기만)
├── 설명 펼치기
├── 일시정지 → 재생
├── 시간 바 앞으로 건너뛰기
└── 화면 아무데나 클릭 (진행바 숨기기)
```

### 4.6 Google 생태계 활용

```
Gmail 확인 (30% 확률):
├── Gmail 접속 → 받은편지함
├── 이메일 1~2개 열어보기 (제목만 or 내용 스크롤)
├── 10~30초 체류
└── YouTube로 복귀

Google 검색 (40% 확률):
├── 페르소나 관심사 키워드로 검색
├── 결과 1~2개 클릭 → 5~15초 체류
├── 뒤로가기 → YouTube로 복귀
└── 검색어 예: "퇴근 후 운동", "점심 맛집", "탈모 예방" 등
```

## 5. 자동 로그인

### 5.1 최초 로그인 (계정 등록 시)

```
1. AdsPower 프로필 열기
2. Google 로그인 페이지 접속
3. 이메일 입력 (type_human)
4. 비밀번호 입력 (type_human)
5. 2FA 코드 입력 (TOTP 시크릿으로 pyotp 생성)
6. 로그인 완료 확인 (프로필 아이콘 존재 확인)
7. 쿠키 자동 저장 (AdsPower 프로필에)
```

### 5.2 세션 시작 시 로그인 상태 확인

```
프로필 열기 → YouTube 접속
  → 우측 상단 프로필 아이콘 확인
  → 로그인 O → 작업 진행
  → 로그인 X → 자동 재로그인 시도
    → 성공 → 작업 진행
    → 캡챠 → 2Captcha 시도 → 실패 시 태스크 실패 + 알림
    → 로그인 실패 → 계정 "login_failed" + 대시보드 알림
```

## 6. 에러 처리

```
에러 유형별 처리:

브라우저 에러:
├── 페이지 로딩 타임아웃 → 재시도 1회 → 실패 시 태스크 실패
├── AdsPower 연결 실패 → 태스크 실패 + 알림
├── Playwright CDP 끊김 → 세션 종료 + 태스크 재배정

YouTube 에러:
├── 캡챠 감지 → 2Captcha 시도 → 실패 시 계정 정지 + 알림
├── "댓글 작성 실패" → 재시도 1회 → 실패 시 태스크 실패
├── 계정 정지 페이지 → 즉시 세션 종료 + 계정 "suspended" + 알림

IP 에러:
├── ADB 연결 실패 → 재시도 3회 → 실패 시 세션 건너뜀
├── IP 변경 실패 → 재시도 3회 → 실패 시 세션 건너뜀

모든 에러:
├── 에러 발생 시 스크린샷 저장 (디버깅용)
├── 에러 로그 서버에 전송
└── 심각한 에러 → 텔레그램 알림
```

## 7. 기존 코드 활용 계획

```
그대로 활용 (복사/임포트):
├── hydra/browser/actions.py — 전체
├── hydra/browser/adspower.py — 전체
├── hydra/browser/driver.py — BrowserSession 클래스
├── hydra/core/behavior.py — 행동 엔진 전체
├── hydra/infra/ip.py — IP 로테이션
├── hydra/ghost/detector.py — 고스트 판정 로직
└── hydra/infra/captcha.py — 2Captcha 연동

재설계 (Worker 태스크 기반):
├── worker/executor.py — 스텁 → 실제 실행기로 교체
├── worker/session.py — 신규, 세션 관리 (열기/닫기/태스크 루프)
├── worker/warmup.py — 신규, 워밍업 전용 실행 로직
└── worker/login.py — 신규, 자동 로그인 + 2FA

사용 안 함 (대체됨):
├── hydra/core/session_runner.py — Worker 세션으로 대체
├── hydra/core/executor.py — Worker executor로 대체
└── hydra/core/scheduler.py — 서버 태스크 분배로 대체
```

## 8. Worker 모델 확장

Worker 테이블에 역할 컬럼 추가:

```python
# Worker 모델에 추가:
allow_preparation = Column(Boolean, default=False)  # 준비 작업 허용
allow_campaign = Column(Boolean, default=True)       # 캠페인 작업 허용
```

서버 태스크 배정 시:
- 로그인/2FA/채널 설정/워밍업 태스크 → `allow_preparation=True`인 Worker에만 배정
- 댓글/좋아요/부스트 태스크 → `allow_campaign=True`인 Worker에 배정
