# HYDRA v2 — UI/UX 재설계 스펙

> 작성일: 2026-04-17
> 상태: 승인됨

## 1. 설계 원칙

토스식 UX — "생각하지 않아도 되게 만드는 것"

1. **화면마다 목표 1개** — 지금 뭘 하면 되는지만
2. **행동 중심 설계** — 설정 → 모니터링 → 대응
3. **시각 계층** — 중요한 것 1개 크게, 나머지 약하게
4. **한 줄에 한 질문** — 입력 필드마다 명확한 라벨 + 설명
5. **문구 = 행동 보조** — "daily_comment_limit" ❌ → "하루에 댓글 몇 개까지 달까요?" ✅
6. **피드백 = 확신** — 저장됐다, 실패했다, 진행 중이다 명확하게
7. **정보는 다 있되, 계층화** — 간편함 ≠ 정보 숨기기. 깊이에 따라 펼쳐보기.

### 컴포넌트
- **버튼 3종:** Primary (퍼플), Secondary (#27272a), Danger (빨강 투명)
- **상태 뱃지:** 배경 투명 + 색상 텍스트 (● 활성, ● 워밍업, ● 쿨다운, ● 정지, 👻 고스트)
- **통계 카드:** 라벨 + 큰 숫자 + 보조 텍스트 (목표 대비 등)
- **입력 폼:** 라벨 + 설명 + 입력, placeholder 대신 설명 텍스트 별도
- **피드백 토스트:** 성공 (초록 테두리 + ✓), 실패 (빨강 테두리 + ✕)
- **페이지네이션:** 숫자 버튼, 한 페이지 20개

## 3. 페이지별 설계

### 3.1 대시보드 (홈)

**목표:** "지금 시스템이 잘 돌아가고 있나?" 한눈에.

**구성:**
- 시스템 상태 배너 (최상단) — 초록/노랑/빨강 LED + "시스템 정상 운영 중" + Worker N대 + 캠페인 N개 + 에러 유무
- 통계 카드 4개 — 오늘 댓글 (목표 대비 %), 오늘 좋아요, 활성 계정 (N/전체), 주의 필요 (빨강, 클릭 이동)
- 진행 중인 캠페인 목록 — 캠페인별 진행률 바 + 댓글/좋아요 카운트 + 어떤 PC 작업 중 + 프리셋 뱃지
- 워커 상태 — 온/오프라인 LED + 이름 + 태스크 수
- 알림 — 에러/완료 이벤트 (빨강 배지 카운트)

### 3.2 브랜드

**목표:** AI가 제품을 이해하기 위한 정보 관리.

**구성:**
- 카드 목록 — 이름 + 카테고리 + 핵심 메시지 요약 + 홍보 키워드 태그
- 생성/수정 폼 (모달) — 4개 필드만:
  - 브랜드 이름
  - 카테고리
  - 핵심 메시지 (셀링 포인트)
  - 홍보 키워드 (태그 입력, Enter로 추가, × 삭제)
- 삭제 버튼 (좌하단, 빨강, 확인 모달)

**브랜드에 없는 것 (캠페인으로 이동됨):**
- 타겟 키워드
- 멘션 스타일
- 프리셋 선택
- 주간 목표
- 자동 캠페인

### 3.3 캠페인

**목표:** 어디에, 어떻게, 얼마나 작업할지. 작업의 모든 것.

**구성:**
- 카드형 캠페인 목록 — 이름 + 상태 뱃지 + 기간 + 진행률 바 + 댓글/좋아요/영상 카운트 + PC 배정
- "캠페인 만들기" 버튼 → 4단계 마법사:
  - Step 1: 브랜드 선택 ("누구의 홍보인가요?")
  - Step 2: 타겟 키워드 입력 ("어떤 영상에 작업할까요?") → 영상 자동 수집
  - Step 3: 프리셋 선택 + 영상당 세트 수 + 멘션 스타일 ("어떻게 작업할까요?")
  - Step 4: 기간 + 목표 영상 수 ("얼마나 할까요?") → 시작
- "다이렉트" 버튼 — URL 입력 + 프리셋 선택 or 수동 입력 (댓글/대댓글/좋아요/구독)
- 캠페인 상세 (클릭 시) — 통계 5개 + 최근 작업 영상 목록 + 일시정지/수정/중단 버튼

**재작업 규칙:**
- 같은 계정 + 같은 영상 = 영구 1회
- 같은 프리셋 + 같은 영상 = 7일 후 (같은 영상에 다른 프리셋은 동시 가능)
- 설정에서 변경 가능

### 3.4 작업 (태스크 큐)

**목표:** 지금 뭐가 돌아가고 있나, 실시간 모니터링.

**구성:**
- 통계 카드 4개 — 대기/진행중/완료/실패
- 태스크 목록 — 상태별 LED 색상 구분:
  - 진행 중: 퍼플 LED + 작업 내용 + 계정 + PC
  - 좋아요 부스트: 진행률 바 (15/30)
  - 대기: 반투명 + 예정 시간
  - 완료: 반투명 + 완료 시각
- 캠페인 필터 드롭다운
- 태스크별 취소 버튼

### 3.5 타겟 (수집 영상)

**목표:** 수집된 영상 확인. 단순 정보 조회.

**구성:**
- 통계 카드 4개 — 전체/완료/대기/마지막 수집 시각
- 영상 테이블 — 제목 + 채널 + 조회수 + 캠페인 + 상태 뱃지
- 캠페인 필터 드롭다운
- URL 수동 추가 버튼
- 페이지네이션 (20개씩)
- 자동 수집은 캠페인에서 설정된 키워드로 백그라운드 실행

### 3.6 분석

**목표:** 성과 확인, 얼마나 효과가 있는지.

**구성:**
- 기간 선택 (7일/30일/전체)
- 통계 카드 4개 — 총 댓글 (추이%) + 총 좋아요 + 작업 영상 + 고스트율
- 일별 작업량 바 차트 (오늘은 점선 테두리)
- 캠페인별 성과 테이블 — 캠페인명 + 댓글 + 좋아요 + 영상 + 고스트

### 3.7 계정 (어드민)

**목표:** 계정 라이프사이클 전체 관리.

**구성:**
- 상태 카드 6개 — 전체/활성/워밍업/쿨다운/정지/고스트 (클릭 필터)
- 3개 탭:
  - 전체 계정: 테이블 (이름 + 페르소나 + 상태 + 오늘 작업 + 성공률 + 마지막 활동)
  - 워밍업: 워밍업 시작 UI (Day 선택 + 계정 체크박스 + 시작 버튼)
    - 표시 대상: registered, Day N 완료 계정만
    - 이미 활성/진행중/정지 → 표시 안 함
  - 문제 계정: 고스트/캡챠/정지 계정 + 일괄 처리
- 행 클릭 → 사이드 패널: 정보 + 통계 + 활동 + 액션 (워밍업 재시작, 상태 변경, 폐기)
- 페이지네이션 (20개씩)

### 3.8 워커 (어드민)

**목표:** Worker PC 관제.

**구성:**
- 통계 카드 3개 — 온라인/태스크/완료
- 워커 카드 (PC당 하나):
  - LED 상태 + 이름 + OS/버전 + 마지막 통신
  - 태스크 수 + 프로필 잠금 수
  - 역할 뱃지 (준비/캠페인)
  - 일시정지/재개 버튼
  - 오프라인: 반투명 + 안내 메시지
- "워커 추가" 버튼 → 이름 + 역할 + 시크릿 → 토큰 발급 (다시 안 보임)

### 3.9 설정 (어드민)

**목표:** 시스템 설정 관리.

**4개 탭:**
- 프리셋: 좌측 프리셋 목록 + 우측 대화 미리보기형 편집기
  - 스텝 카드: 번호 + 역할 뱃지 + 대상 + 톤 설명 + 좋아요 수
  - 들여쓰기로 대댓글 구조 시각화
  - 스텝 사이에 딜레이 표시 (↓ 5~25분 후)
  - 스텝 클릭 → 편집 모달 (역할, 유형, 톤, 대상, 좋아요, 딜레이)
  - + 스텝 추가, 삭제, 복제
- 행동 패턴: 일일/주간 한도, 쿨다운, 세션 간격 (토스식 폼)
- 일반: API 키 (Claude, YouTube, 2Captcha), 텔레그램 알림
- 외관: 테마 전환

## 4. 구조 변경 사항 (사양서 반영)

### 브랜드 ↔ 캠페인 역할 분리

**브랜드 (변경):**
- 순수 브랜드 정보만 (이름, 카테고리, 핵심 메시지, 홍보 키워드)
- 타겟 키워드, 멘션 스타일, 프리셋, 주간 목표 제거

**캠페인 (변경):**
- 브랜드 선택
- 타겟 키워드 (영상 검색용) — 브랜드에서 이동
- 멘션 스타일 (제품 언급 방식) — 브랜드에서 이동
- 프리셋 선택 — 브랜드에서 이동
- 영상당 프리셋 세트 수 (1~N개)
- 작업 계획 (기간 + 목표 수량)

### 재작업 규칙 변경
- 기존: 같은 브랜드 + 같은 영상 = 7일 차단
- 변경: 같은 프리셋 + 같은 영상 = 7일 차단 (다른 프리셋은 동시 가능)
